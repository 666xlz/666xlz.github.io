# 卡牌桌游系统 - 开发文档

## 一、项目概述

### 1.1 项目名称
卡牌桌游在线管理系统

### 1.2 项目定位
一款基于 Web 的卡牌桌游管理工具，支持主持人创建房间、玩家加入、卡牌管理、战场模式等功能。

### 1.3 技术架构
- **前端技术**：原生 HTML + CSS + JavaScript
- **实时通信**：PeerJS 1.5.4（WebRTC）
- **本地存储**：IndexedDB（图片）+ localStorage（配置）
- **无需后端**：纯前端应用，依赖 PeerJS Cloud Server

### 1.4 文件结构
```
666/
├── index.html      # 首页（角色选择）
├── host.html       # 主持人端
├── host.js         # 主持人端逻辑
├── player.html     # 玩家端
├── player.js       # 玩家端逻辑
├── app.js          # 本地模式逻辑（单人）
├── admin.html      # 激活码管理页面
├── style.css       # 全局样式
└── README.md       # 项目说明
```

## 六、已知限制与注意事项

### 6.1 技术限制
1. **PeerJS 依赖**：需要访问 `unpkg.com`，离线环境无法使用
2. **IndexedDB 容量**：浏览器限制，一般 50MB-无限制
3. **图片压缩**：单张建议不超过 100KB
4. **房间号**：6位字母数字，最多约 20 亿组合

### 6.2 同步问题
1. 手牌图片在切换标签时需重新加载
2. 战场背景图同步传输可能较慢
3. 多设备同时操作需主持人协调

### 6.3 安全注意
- 激活码验证在前端，存在被破解风险
- 敏感数据存储在 localStorage，仅适合非商业场景
- 无权限控制，任何知道房间号的人都能加入
