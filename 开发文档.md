# 卡牌桌游系统开发文档

> 更新时间：2026年4月19日

## 一、系统概述

这是一个基于 Web 的在线桌游卡牌管理系统，支持多人联机游戏。系统包含多个入口页面：
- `index.html` - 身份选择入口（主持人/玩家选择）
- `host.html` - 主持人端（DM端）
- `player.html` - 玩家端
- `app.js` - 通用本地桌游逻辑（可整合到其他页面）

### 技术架构
- **通信**：PeerJS (基于 WebRTC 的 P2P 通信)
- **前端**：原生 HTML/CSS/JavaScript
- **状态管理**：localStorage 本地持久化
- **界面风格**：QQ风格（蓝色主题，圆角设计）

## 二、主持人端 (host.html/host.js)

### 2.1 注册登录系统
| 功能 | 描述 |
|------|------|
| **注册** | 输入授权码 + 设置名称 + 设置密码 |
| **登录** | 输入名称和密码登录 |
| **一人一码** | 每个授权码只能注册一个账号 |
| **退出登录** | 退出当前账号，返回登录界面 |

> **授权码管理**：管理员分发唯一授权码给用户，用户凭授权码注册账号后，授权码即被使用

### 2.2 房间管理
| 功能 | 描述 |
|------|------|
| 创建房间 | 生成唯一房间号，弹窗显示供玩家加入 |
| 加入房间 | 通过房间号加入其他主持人的房间 |
| 刷新显示 | 刷新所有界面显示 |
| 清理状态 | 重置所有游戏状态 |

### 2.3 卡池管理
| 功能 | 描述 |
|------|------|
| 创建卡池 | 输入名称创建新卡池 |
| 删除卡池 | 自定义确认弹窗确认后删除整个卡池及卡牌 |
| 添加卡牌 | 上传图片(多选)到指定卡池，图片文件名作为默认名称 |
| 删除卡牌 | 从卡池移除单张卡牌 |
| 查看卡池 | 显示所有卡池及卡牌数量 |
| **卡池标签页** | 多个卡池用标签切换显示 |

### 2.4 弃牌堆管理
| 功能 | 描述 |
|------|------|
| 查看弃牌堆 | 显示所有弃置的卡牌 |
| 选择模式 | 点击"选择"按钮进入多选模式 |
| 移动到卡池 | 将选中的弃牌通过下拉选择框移回到指定卡池 |
| 取消选择 | 退出选择模式 |

### 2.5 玩家管理
| 功能 | 描述 |
|------|------|
| 查看在线玩家 | 列表显示所有已连接玩家，标注在线/空闲状态 |
| 查看玩家手牌 | 选择玩家查看其当前手牌 |
| **踢出玩家** | 断开在线玩家的连接，但保留角色和手牌供其他玩家选择 |
| **删除角色** | 删除角色及其手牌（在线玩家会先被踢出） |
| **角色互斥选择** | 记录每个玩家选择的角色，防止重复选择 |
| **玩家退出处理** | 玩家断开时自动释放角色记录 |

### 2.6 牌桌
| 功能 | 描述 |
|------|------|
| 显示出牌 | 显示所有玩家出的牌（含玩家名称） |
| 布局 | CSS Grid，每行8张卡牌 |

### 2.7 图片功能
| 功能 | 描述 |
|------|------|
| 放大查看 | 所有区域的图片均可点击放大 |
| 模态框 | 深色背景，图片宽度500px（最大95vw） |
| 关闭方式 | 点击X按钮、点击背景、ESC键 |

## 四、网络通信机制

### 4.1 消息类型
| 类型 | 方向 | 描述 |
|------|------|------|
| `sync` | Host → Player | 同步游戏状态（卡池、弃牌堆、玩家手牌等） |
| `select_player` | Player → Host | 玩家选择角色请求 |
| `select_result` | Host → Player | 角色选择结果（成功/失败） |
| `quit_player` | Host → All | 玩家退出/断开通知 |
| `card_action` | Player ↔ Host | 卡牌操作（抽卡、使用、弃牌等） |
| `give_card` | Player ↔ Host | 赠送卡牌 |
| `broadcast` | Host → All | 广播消息 |

### 4.2 状态同步
- 主持人端（Host）维护完整游戏状态
- 玩家连接时发送 `sync` 数据同步状态
- 玩家状态变化实时同步给主持人
- **角色选择互斥**：`broadcastData` 包含 `takenPlayerIds` 数组，标识已被占用的角色

### 4.3 玩家连接管理
```javascript
playerConnections = {
    [playerId]: {
        name: string,
        playerName: string,
        peerId: string
    }
}
```

## 六、UI/UX 规范

### 6.1 QQ风格颜色主题
| 用途 | 颜色代码 |
|------|----------|
| 主色/强调色 | `#0095f6` (QQ蓝) |
| 背景色 | `#f5f5f5` (浅灰) |
| 卡片背景 | `#ffffff` (白色) |
| 文字主色 | `#333333` (深灰) |
| 次要文字 | `#999999` (浅灰) |
| 边框/分隔线 | `#e5e5e5` |

### 6.2 布局规范
- 卡牌尺寸：100x140px (宽x高)
- 卡牌圆角：10px
- 按钮圆角：6px
- 模态框圆角：16px
- 区域间距：12px

### 6.3 模态框规范
- 背景：`rgba(0,0,0,0.6)`
- 内容宽度：最大500px（图片展示），最小300px（确认弹窗）
- 图片：宽度100%，最大高度85vh
- 关闭按钮：白色，顶部右侧
- 关闭方式：点击X、点击背景、ESC键

### 6.4 日志颜色规范
| 日志类型 | 颜色 | CSS类 |
|----------|------|-------|
| 抽卡 | 绿色 | `.log-draw` |
| 使用卡牌 | 蓝色 | `.log-use` |
| 弃牌 | 红色 | `.log-discard` |
| 系统消息 | 灰色 | `.log-system` |
| 玩家消息 | 橙色 | `.log-player` |
| 信息提示 | 青色 | `.log-info` |
| 奖励 | 紫色 | `.log-reward` |
| 警告 | 黄色 | `.log-warning` |

### 6.5 自定义组件
- **confirmModal**：自定义确认弹窗，替代浏览器原生 `confirm()`
- **下拉选择框**：用于弃牌堆回收等场景，替代 `prompt()`
- **标签页切换**：卡池管理中使用标签页切换不同卡池
